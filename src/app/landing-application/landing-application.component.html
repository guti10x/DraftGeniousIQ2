Fases de la aplicación

Fase 1: Investigación y Adquisición de Datos

Iniciamos nuestro proyecto con una fase de investigación exhaustiva, donde la selección de fuentes confiables fue primordial. Seleccionamos la API de Sofascore por su integridad y la calidad superior de datos que ofrece, elementos esenciales para el análisis preciso en el contexto de ligas fantasy de fútbol. La confiabilidad de los datos es la piedra angular de nuestro sistema y fundamenta la exactitud de nuestras proyecciones y análisis.

Fase 2: Extracción Avanzada de Datos

Para complementar nuestra base de datos, implementamos técnicas de web scraping sofisticadas. Se llevó a cabo la extracción sistemática de datos desde una cuidadosa selección de sitios web, lo cual nos permitió acumular un conjunto de datos más robusto y diverso para fortalecer los análisis posteriores.

Fase 3: Minería y Modelado de Datos

En esta etapa crucial, consolidamos los datos recolectados en varios datasets. Comenzamos el proceso de minería de datos, una exploración profunda que nos permitió entender las tendencias y patrones subyacentes. Tras un proceso de evaluación rigurosa, seleccionamos y calibramos los modelos analíticos más prometedores, optimizándolos para nuestro uso específico. Los modelos finalistas fueron elegidos por su precisión predictiva y relevancia para las necesidades de nuestros usuarios.

Fase 4: Desarrollo de la Interfaz de Usuario y Pruebas

Con los modelos analíticos en su lugar, procedimos a la fase de implementación. Esto incluyó el entrenamiento intensivo de los modelos con nuestro dataset de alta calidad y la realización de pruebas exhaustivas para garantizar su fiabilidad. Paralelamente, nos dedicamos a la creación de una primera interfaz de usuario algo primitiva, diseñada para facilitar el acceso a nuestros análisis.

Fase Actual: Desarrollo de la Estructura de Base de Datos y Diseño de la Interfaz Preliminar

Nos encontramos en una fase decisiva de nuestro proyecto que implica el diseño meticuloso de la estructura de la base de datos (BD) y el establecimiento de una interfaz de usuario básica. Nuestro objetivo es crear una infraestructura robusta que soporte eficientemente la recolección, almacenamiento y recuperación de datos analíticos.
